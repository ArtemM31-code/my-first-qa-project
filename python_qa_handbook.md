# 🟢 ТЕМА 1: Переменные и типы данных

> **Теория:** Переменная — это не коробка, а "стикер", который Питон наклеивает на ячейку в оперативной памяти компьютера. Когда мы пишем `price = 100`, Питон кладет `100` в память и клеит на неё ярлык `price`. Если мы пишем `price2 = price`, создается второй ярлык на ту же самую ячейку.

### 💻 Пример 1: Базовые типы данных
```python
user_name = "Artem"      # Строка (String)
user_age = 25            # Целое число (Integer)
balance = 150.50         # Дробное число (Float)
is_admin = True          # Логический тип (Boolean)

print(f"Пользователь {user_name}, возраст {user_age}. Админ: {is_admin}")
```

### 💻 Пример 2: Перезапись переменных (Смена стикеров)
В тестировании мы часто перезаписываем одну и ту же переменную, когда переходим по страницам сайта.
```python
current_url = "https://mysite.com/login"
print("Мы находимся на:", current_url)

# Перешли на другую страницу
current_url = "https://mysite.com/dashboard" 
print("Теперь мы тут:", current_url)
```

### 💻 Пример 3: Преобразование типов
Часто из файлов или API данные приходят в виде текста, даже если это цифры. Их нужно конвертировать.
```python
age_from_api = "30" # Это строка (текст)!
age_number = int(age_from_api) # Превратили текст в настоящее число

if age_number > 18:
    print("Доступ разрешен")
```

# 🟡 ТЕМА 2: Управление потоком (Условия и Циклы)

> **Теория:** Код всегда выполняется сверху вниз. Условия (`if / else`) заставляют код ветвиться в зависимости от ситуации. Циклы (`while`, `for`) заставляют код повторять одно и то же действие много раз.

### 💻 Пример 1: Сложные условия (if / elif / else)
```python
http_status = 500

if http_status == 200:
    print("Успех! Сайт работает.")
elif http_status == 404:
    print("Ошибка 404: Страница не найдена.")
elif http_status == 500:
    print("Ошибка 500: Сервер упал!")
else:
    print("Неизвестный статус.")
```

### 💻 Пример 2: Цикл `while` для ожидания (Waits)
Используется, когда мы не знаем, сколько раз придется повторить действие. Идеально для ожидания загрузки элементов в автотестах.
```python
is_button_visible = False
attempts = 0

while is_button_visible == False and attempts < 5:
    print("Ждем появления кнопки...")
    attempts += 1
    # Допустим, на 3-й попытке кнопка загрузилась и появилась
    if attempts == 3:
        is_button_visible = True

print("Кнопка найдена! Кликаем.")
```

### 💻 Пример 3: Цикл `for` для массовых проверок
Используется, когда у нас уже есть готовый список данных.
```python
users_to_check = ["admin", "moderator", "guest"]

for user in users_to_check:
    print(f"Проверяем права для пользователя: {user}")
```

# 🔵 ТЕМА 3: Структуры данных и Изменяемость

> **Теория:** Строки и числа **неизменяемы** (создаются заново). Списки (`List`) и словари (`Dictionary`) **изменяемы** (меняются изнутри без создания новых ячеек в памяти). Это частый вопрос на собеседованиях.

### 💻 Пример 1: Словари (Dictionary) — работа с API JSON
Словари состоят из ключей и значений. Все ответы от серверов в вебе (JSON) работают как словари.
```python
api_response = {
    "user_id": 105,
    "email": "test@mail.com",
    "is_active": True
}

# Достаем данные по ключу
if api_response["is_active"]:
    print(f"Отправляем письмо на {api_response['email']}")
```

### 💻 Пример 2: Списки (List) — динамическое добавление
```python
cart_items = ["Телефон"]
print("До добавления:", cart_items)

# Пользователь кликнул "Добавить в корзину"
cart_items.append("Чехол")
cart_items.append("Стекло")

print("После добавления:", cart_items)
```

### 💻 Пример 3: Множества (Set) — удаление дубликатов
Множества мгновенно уничтожают любые повторяющиеся элементы. Полезно для фильтрации баз данных.
```python
clicked_buttons = ["login_btn", "login_btn", "cancel_btn", "login_btn"]
unique_clicks = set(clicked_buttons)

print("Все клики:", clicked_buttons)
print("Уникальные элементы:", unique_clicks) # Выведет {'login_btn', 'cancel_btn'}
```

# 🟣 ТЕМА 4: Функции и Области видимости

> **Теория:** Функция — это мини-завод. Аргументы (в скобках) — это сырье. `return` — это готовая продукция. Переменные внутри функции умирают сразу после её завершения (Локальная область видимости).

### 💻 Пример 1: Зачем нужен `return`?
Если мы используем `print()`, программа просто выведет текст на экран и забудет его. `return` позволяет вытащить данные в основную программу.
```python
def calculate_discount(price, discount_percent):
    final_price = price - (price * discount_percent / 100)
    return final_price  # Отдаем результат наружу!

# Сохраняем результат в переменную
iphone_price = calculate_discount(1000, 20)
print(f"Цена со скидкой: {iphone_price}$")
```

### 💻 Пример 2: Переиспользование функций (Генератор данных)
В тестах нам постоянно нужны уникальные данные. Функция помогает не дублировать код.
```python
import random

def generate_random_email():
    random_number = random.randint(100, 999)
    return f"qa_test_{random_number}@gmail.com"

print("Email 1:", generate_random_email())
print("Email 2:", generate_random_email())
```

### 💻 Пример 3: Локальная переменная (Ошибка видимости)
```python
def make_payment():
    receipt_number = "12345-ABC" # Переменная умрет после завершения функции
    print("Оплата прошла успешно!")

make_payment()
# print(receipt_number) # ОШИБКА! Главная программа не видит эту переменную.
```

# 🟠 ТЕМА 5: Работа с файлами (Менеджер контекста)

> **Теория:** Конструкция `with open(...)` гарантирует, что файл будет безопасно закрыт, даже если программа упадет с критической ошибкой. В тестах мы постоянно читаем конфигурации и пишем отчеты.

### 💻 Пример 1: Режим "w" (Write) — полная перезапись
```python
# Режим "w" уничтожит старое содержимое и запишет новое
with open("report.txt", "w", encoding="utf-8") as file:
    file.write("--- НАЧАЛО ОТЧЕТА ---\n")
    file.write("Все тесты пройдены успешно!\n")
```

### 💻 Пример 2: Режим "a" (Append) — тихое добавление логов
Режим `a` часто используется, когда множество тестов по очереди дописывают свои результаты в один общий файл.
```python
test_name = "Auth_Test_01"
status = "PASSED"

with open("report.txt", "a", encoding="utf-8") as file:
    file.write(f"Тест: {test_name} | Статус: {status}\n")
```

### 💻 Пример 3: Режим "r" (Read) с обработкой ошибок
Например, когда мы вынесли все логины/пароли в текстовый файл, чтобы не хардкодить их в скрипте.
```python
try:
    with open("passwords.txt", "r", encoding="utf-8") as file:
        passwords_list = file.read()
        print("Загруженные пароли:\n", passwords_list)
except FileNotFoundError:
    print("Ошибка: Файл с паролями не найден!")
```

# 🟤 ТЕМА 6: Объектно-Ориентированное Программирование (ООП)

> **Теория:** ООП — это подход, где программа состоит не из команд, а из **Объектов**. 
> **Класс** — это чертеж (например, чертеж кнопки). 
> **Объект** — это реальная сущность, созданная по чертежу (реальная Зеленая кнопка "Войти").
> В автотестах мы 90% времени ИСПОЛЬЗУЕМ объекты готовых классов, которые написали разработчики фреймворков.

### 💻 Пример 1: Использование готового Класса
Чтобы создать объект, нужно написать имя Класса и передать ему свойства в скобках.
```python
# Создаем реальную кнопку по чертежу Button
login_btn = Button("Войти", "Синяя") 

# Вызываем действие (Метод) у этой кнопки через точку
login_btn.click()
```

### 💻 Пример 2: Класс Page Object (Основа тестирования UI)
В тестах мы описываем страницы сайта как Классы.
```python
# Так мы создаем класс страницы
class LoginPage:
    def __init__(self, browser):
        self.url = "https://mysite.com/login"
        self.browser = browser

    def login_as_admin(self):
        print("Вводим админский логин и пароль...")

# Так мы используем его в тесте
page = LoginPage(chrome)
page.login_as_admin()
```

# 🔴 ТЕМА 7: Базовое тестирование с Pytest

> **Теория:** Pytest — это инструмент, который автоматически находит и запускает функции, чье название начинается с `test_`. Главное оружие тестировщика — команда `assert` (утверждаю).

### 💻 Пример 1: Утверждение истины
Если после `assert` стоит ложь (False), Pytest зажжет красную лампочку (FAILED) и выдаст подробный отчет с указанием строки ошибки.
```python
def test_math():
    assert 2 + 2 == 4
    
def test_age():
    age = 15
    assert age >= 18 # ТЕСТ УПАДЕТ ЗДЕСЬ!
```

# 🔴 ТЕМА 8: Продвинутые assert-проверки в реальных условиях

> **Теория:** В реальном автотестировании (сайтов или API) мы редко проверяем просто числа. Чаще всего мы ищем ключевые слова в текстах страниц или проверяем количество товаров. Для этого используются оператор `in` и встроенная функция `len()`.

### 💻 Пример 1: Проверка вхождения текста (Оператор `in`)
Помогает убедиться, что на странице сайта появился нужный текст (например, сообщение об ошибке).
```python
def test_error_message():
    site_text = "Ошибка: Ваш аккаунт заблокирован за спам!"
    
    # Проверяем, есть ли подстрока "заблокирован" внутри общего текста
    assert "заблокирован" in site_text
```

### 💻 Пример 2: Проверка длины списка/словаря (Функция `len()`)
Позволяет проверить, что в корзину добавилось нужное количество товаров или API вернуло непустой список пользователей.
```python
def test_cart_items():
    cart = ["Ноутбук", "Мышка", "Коврик"]
    
    # Проверяем, что в списке ровно 3 товара
    assert len(cart) == 3
```

# 🔴 ТЕМА 9: Фикстуры (Fixtures) в Pytest

> **Теория:** Фикстура — это функция-помощник, которая выполняет подготовительные действия (Setup) ДО начала теста и уборочные действия (Teardown) ПОСЛЕ окончания теста. 
> Вместо `return` в фикстурах используется ключевое слово `yield`, которое делит работу фикстуры на "до" и "после" теста.

### 💻 Пример 1: Фикстура для эмуляции браузера
```python
import pytest

@pytest.fixture
def browser():
    # Всё, что ДО yield — это подготовка (выполняется перед тестом)
    print("\n[SETUP] Открываем браузер Chrome...")
    browser_driver = "Chrome Browser Instance"
    
    yield browser_driver  # Отдаем объект браузера в тест
    
    # Всё, что ПОСЛЕ yield — это уборка (выполняется после теста)
    print("\n[TEARDOWN] Закрываем браузер и чистим кэш...")

# Тест принимает фикстуру в качестве аргумента по её имени!
def test_search(browser):
    print(f"Выполняем тест в браузере: {browser}")
    assert "Chrome" in browser
```

# 🔵 ТЕМА 10: UI Автоматизация с Playwright

> **Теория:** Playwright — это современная библиотека для управления реальными браузерами. Он позволяет роботу заходить на сайты, кликать по кнопкам, заполнять формы ввода и делать скриншоты.

### 💻 Пример 1: Тест поиска в Википедии
```python
from playwright.sync_api import sync_playwright

def test_wikipedia_search():
    # Запускаем менеджер Playwright
    with sync_playwright() as p:
        # Открываем реальный браузер Chromium
        # headless=False показывает окно браузера на экране
        # slow_mo=1000 замедляет робота на 1 секунду, чтобы человек успевал следить за кликами
        browser = p.chromium.launch(headless=False, slow_mo=1000)
        
        # Создаем новую вкладку
        page = browser.new_page()
        
        # Переходим на сайт
        page.goto("https://ru.wikipedia.org/")
        
        # Заполняем поле поиска (находим его по селектору input[name='search'])
        page.fill("input[name='search']", "Python")
        
        # Нажимаем клавишу Enter в этом же поле
        page.press("input[name='search']", "Enter")
        
        # Ждем загрузки и проверяем заголовок вкладки
        page.wait_for_load_state("load")
        title = page.title()
        
        # Assert: проверяем, что слово Python есть в заголовке вкладки
        assert "Python" in title
        
        # Закрываем браузер
        browser.close()
```
